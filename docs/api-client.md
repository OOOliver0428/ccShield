# Frontend ↔ backend API client

The FastAPI backend exposes its full surface as an OpenAPI 3.x document at
`GET /openapi.json`. From that document we generate a typed TypeScript SDK
into `frontend/src/api/generated/`. The directory is committed; the
`.github/workflows/ci.yml` job `openapi-stale` re-runs the generator on
every PR and fails if the committed artifacts drift.

## Source of truth for the contract

`frontend/src/api/generated/` is the *source of truth* for the API shape:
every field name, every path parameter, every response code. The hand-written
`frontend/src/api/client.ts` exists because it is what the MVP actually
calls today (single-user local bootstrap, INTERCEPTOR-driven bearer
token); it predates the generator and is NOT regenerated by these scripts.

Starting with v2.0.3, `POST /api/ban` accepts only the duration values exposed by the
current B站 live-room UI: `-1` (permanent), `0` (current stream), and
`2 / 4 / 24 / 168` hours. The generated `BanRequest.hour` union and
`frontend/openapi.json` must be updated together whenever this contract changes.

## Regeneration workflow

After you change a backend route, parameter, or response model:

```bash
bash scripts/gen_schema.sh     # boots uvicorn, fetches /openapi.json
bash scripts/gen_client.sh     # regenerates frontend/src/api/generated/
git add frontend/openapi.json frontend/src/api/generated/
git commit -m "build(api): regenerate client for <change>"
```

Both scripts are idempotent and headless. `scripts/gen_schema.sh` exits
non-zero if the backend fails to start or `/openapi.json` is unreachable;
it refuses to bind a port already in use (so it will not clobber a
running dev server — override with `CCSHIELD_GEN_PORT=<free>`).

`scripts/gen_client.sh` requires `@hey-api/openapi-ts`, declared as a
frontend devDependency (so CI's `bun install --frozen-lockfile` resolves
it deterministically).

## Migration to the generated SDK (follow-up)

The generated `client()` function and `sdk.gen.ts` operations are the
intended replacement for `frontend/src/api/client.ts` once the MVP is
out of the way. Future migration plan:

1. Replace `httpClient.get('/auth/bootstrap')` with the generated
   `bootstrap({ ... })` operation, swapping the axios interceptor
   for the generated client's `auth` hook.
2. Replace the per-call `axios.get('/api/...')` sites with the typed
   SDK functions or `client.get<...>({ url: '/...' })`.
3. Delete `client.ts` once no callers remain.
4. Confirm CI's `openapi-stale` job still passes — backend route
   renames without regen will already fail the gate.

The CI stale gate is what makes this safe to defer: as long as it
runs, contract drift between `client.ts` and the live backend
becomes impossible to merge by accident.
