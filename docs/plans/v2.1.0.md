# ccShield v2.1.0 发布方案

> 状态：已冻结，进入正式发布
> 更新日期：2026-07-16

## 版本目标

v2.1.0 是 ccShield 首个同时面向开发者和普通 Windows 用户的正式发行版本。在 v2.0.4 已完成的弹幕字号、房间身份、Cookie 失效闭环等能力之上，本版本补齐免开发环境的下载、启动、数据隔离、成品验证和 GitHub Release 发布链路。

## 已冻结的产品能力

- 普通实时弹幕支持 12、14、16、18px 四档字号，设置保存在浏览器本地。
- 调整字号时保留实时跟随或历史审阅位置，SC 保持固定单行横向滚动和价格降序。
- 连接房间后展示当前登录用户的主播、房管、观众或暂未识别身份。
- Cookie 运行中失效时停止旧房间与名单同步、清理工作台并引导重新扫码。
- 延续 1000 条弹幕缓存、醒目留言审阅、快捷房间和基础禁言闭环。

## Release 实现

- 首发平台：Windows x64 免安装压缩包。
- 运行时：Python 3.11 + PyInstaller `onedir`。
- 前端：生产构建后由 FastAPI 同源提供，Release 不运行 Vite。
- 启动：双击 `ccShield.exe`，自动选择 8000 起的可用端口并打开浏览器。
- 用户数据：统一保存在 `%LOCALAPPDATA%\ccShield`；包含 `.env`、快捷房间和日志。
- 发布：推送 `v2.1.0` 标签后，由 GitHub Actions 构建、冒烟测试并创建 Release。
- 成品：`ccShield-v2.1.0-windows-x64.zip` 和对应 `.sha256` 校验文件。

## 已确认的范围和限制

- Linux 与 macOS 本版继续使用一键源码启动脚本，不提供二进制包。
- Windows 成品暂不进行代码签名，首次运行可能出现 SmartScreen 提示。
- 不自动迁移源码运行时的 `.env` 或快捷房间；Release 首次启动重新扫码登录，避免误搬运个人凭据。
- 发行版仍是本地 Web 应用，不改变既有房管操作的二次确认与安全边界。

## 发布门槛

- Ruff、Basedpyright、后端测试及覆盖率全部通过。
- 前端 lint、类型检查、测试和生产构建全部通过。
- 深色/浅色、常规横屏和电脑版竖屏完成视觉检查。
- Windows 打包成品通过 `/health`、首页和本地数据隔离冒烟测试。
- 自动化测试不得连接真实直播间或执行禁言、解禁等写操作。
- 程序、前端、后端、标签与 Release 文件名统一为 `2.1.0`。
