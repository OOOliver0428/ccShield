# ccShield v2.1.0 规划草案

> 状态：持续调整中
> 更新日期：2026-07-16
> 当前决定：先完成产品功能迭代，暂不实施 Release 打包。

## 版本目标

v2.1.0 作为下一个功能版本，将在 v2.0.4 的弹幕审阅体验基础上继续迭代，并为不具备开发环境的普通用户准备正式下载包。版本号在功能冻结、完整测试通过并准备发布时统一更新；开发过程中保持修订版本号。

## 已在 v2.0.4 完成的基础能力

### 实时弹幕字号

- 仅调整普通实时弹幕，当前 12px 作为最小档，支持 12、14、16、18px 四档。
- 正文、用户名、时间和徽章随档位适度调整；SC、模块标题和房管按钮不跟随缩放。
- SC 继续保持固定单行横向滚动和价格降序，不因字号变化挤压或换行。
- 调整字号时保留实时跟随或历史审阅状态，设置保存在浏览器本地。

### 房间身份

- 房间连接成功后展示当前登录用户的房间身份：主播、房管或观众。
- 主播通过登录 UID 与房间主播 UID 比对；房管通过只读管理员名单分页匹配。
- 查询失败显示“暂未识别”，不能误判为观众，也不能影响弹幕连接。

## v2.1.0 发布前待定功能

- 根据 v2.0.4 的实际使用反馈继续优化弹幕字号、密度和审阅布局。
- Release 打包前允许继续追加和调整 2.1.0 功能，最终以功能冻结时的需求为准。
- Release 打包前新增的产品需求在功能冻结时统一确认，本草案不提前锁死功能范围。

## Release 候选方案（暂缓实施）

- 首发候选为 Windows x64 免安装压缩包，用户无需安装 Python、uv、Node 或 Bun。
- 前端构建为静态资源并由 FastAPI 同源提供，Release 模式不运行 Vite。
- 候选打包技术为 Python 3.11 + PyInstaller `onedir`，文件名暂定 `ccShield-v2.1.0-windows-x64.zip`。
- 双击程序后自动选择本地可用端口、打开浏览器，并提供打开页面、打开数据目录和停止退出入口。
- 候选用户数据目录为 `%LOCALAPPDATA%\ccShield`；登录 Cookie、快捷房间和日志都不得进入安装包或 Git 仓库。
- 标签 `v2.1.0` 触发 GitHub Release 构建、成品冒烟测试和 SHA-256 校验文件生成。

## 正式打包前必须复核

- 是否仍只发布 Windows x64，或增加 macOS/Linux。
- 使用小型控制窗口、命令窗口还是系统托盘。
- 用户数据最终保存到系统用户目录还是程序旁的 `data` 目录。
- 是否具备 Windows 代码签名条件，以及未签名程序的 SmartScreen 提示说明。
- 是否需要从源码运行版本自动迁移 `.env` 和快捷房间。

## 发布门槛

- Ruff、Basedpyright、后端测试及覆盖率全部通过。
- 前端 lint、类型检查、测试和生产构建全部通过。
- 深色/浅色、常规横屏和电脑版竖屏完成视觉检查。
- 打包成品通过首页、健康检查、登录和本地数据隔离的冒烟测试。
- 自动化测试不得连接真实直播间或执行禁言、解禁等写操作。
