# 更新日志

本项目遵循 `主版本.次版本.修订版本` 的版本格式。

## 未发布

## v2.1.0 — 2026-07-16

### 仓库治理

- 将本地开发代理工作目录和运行日志移出版本控制并加入永久忽略规则，本地文件不会被删除。
- 补充贡献指南、安全策略、社区行为准则、Issue 表单和 Pull Request 模板。
- 重构 README 为产品入口、快速开始、文档导航和贡献入口，并增加 CI 与技术栈状态徽章。
- 增加 `.editorconfig` 与 `.gitattributes`，统一跨平台编辑格式和启动脚本行尾。
- 统一遗留的 `reccshield` 包名、标题和内部说明为 `ccShield`，移除内部任务编号与一次性工作报告引用。
- 增加 MIT License，并在 README 中明确授权方式。

### 修复

- Cookie 在运行中失效时统一返回 `BILI_AUTH_EXPIRED`，不再将后续房管请求泄漏为 500 或模糊错误。
- 房间连接、禁言、解禁、名单刷新和后台名单同步均可触发登录过期闭环。
- 登录过期后停止当前房间及禁言名单任务，前端关闭旧 WebSocket、清理工作台并自动进入扫码登录页。
- 多个并发请求同时发现过期时只执行一次前端切换，不会中断新二维码的轮询。

### 安全与约束

- 过期前失败的禁言或解禁请求不会在重新登录后自动重试，避免重复处罚或恢复操作。
- 所有回归测试均使用 Mock 响应，不连接真实 B站，也不执行禁言、解禁等写操作。

### 新增

- 发布首个面向普通用户的 Windows x64 免安装压缩包，双击即可启动本地服务并打开管理页面。
- Release 使用 FastAPI 同源提供已构建前端，不再要求用户安装或运行 Vite、Python、Node.js、Bun 或 uv。
- Release 模式将 Cookie、快捷房间与日志隔离保存到 `%LOCALAPPDATA%\ccShield`，不会写回程序目录或 Git 仓库。
- 标签 `v2.1.0` 触发 Windows 成品构建、打包后冒烟测试、SHA-256 校验文件生成和 GitHub Release 发布。

### 改进

- 启动器会从 8000 起自动选择可用端口，并提供重新打开页面、打开数据目录和安全停止入口。
- 增加跨组件版本一致性检查，防止后端、前端、启动器和发布标签版本不一致。
- 保留 Windows、Linux 与 macOS 的源码启动方式，Release 首发不改变现有开发流程。

### 发布安全与约束

- 自动化发行测试只检查本地健康接口和静态页面，不连接真实直播间，也不执行禁言、解禁等写操作。
- Windows 成品暂未进行代码签名，发布说明明确提示 SmartScreen 风险及校验方式。

## v2.0.4 — 2026-07-16

### 新增

- 实时弹幕新增 12、14、16、18px 四档字号调节，当前 12px 保持为最小档。
- 字号选择保存在浏览器本地，下次打开页面时自动恢复。
- 连接直播间后展示当前登录用户在该房间的身份：主播、房管、观众；只读权限查询失败时显示“暂未识别”。

### 改进

- 普通弹幕正文、用户名、时间、徽章、头像与行高随字号档位协调调整，SC、模块标题和房管操作保持原尺寸。
- 调整字号时保留实时跟随状态；历史审阅状态下锚定首条可见弹幕，减少内容跳动。
- SC 继续保持固定单行横向滚动和价格降序，不因普通弹幕字号变化挤占审阅空间。

### 安全与约束

- 在现有 `.gitignore` 基础上增加 CI 硬性检查，禁止真实 `.env` 和 `config/quick_rooms.json` 进入 Git 历史。
- v2.1.0 的 Windows 免安装 Release 方案先保存为规划草案，完成前置功能迭代后再复核实施。

## v2.0.3 — 2026-07-15

### 新增

- 开放永久禁言，并提供本场、2 小时、4 小时、24 小时、7 天和永久六档期限。
- 禁言成功后显示 3 秒自动消失的非阻塞提示，无需管理员再次确认。
- 禁言名单新增操作管理员名称与 UID 展示；B站未返回该字段时显示为空状态。
- 实时弹幕进入历史审阅状态后累计新消息数量，并提供“查看最新弹幕”快捷按钮。

### 改进

- 实时弹幕缓存上限由 500 条提升至 1000 条。
- 用户向上滚动鼠标或拖动滚动条时暂停自动跟随；恢复最新位置后继续实时滚动。
- 弹幕行改用稳定标识，达到缓存上限淘汰旧消息时减少审阅内容跳动。
- 醒目留言改为固定单行横向滚动，按价格从高到低排列，不再挤占普通弹幕审阅空间。

### 修复

- 按 B站网页端契约修复禁言参数：本场使用 `type=2, hour=0`，定时与永久使用 `type=1`。
- 修复定时禁言错误使用不兼容接口导致的 `参数错误`。
- 修复普通弹幕毫秒时间戳被重复放大、显示时间错乱的问题。
- 完善禁言名单字段兼容和操作管理员信息在缓存、接口及 WebSocket 中的传递。

### 安全与约束

- 自动化测试全部使用合成响应、MockTransport 和前端 HTTP Mock，不执行真实禁言或解禁。
- 永久禁言保留明确期限提示与二次确认；解禁仍只能从禁言名单中基于 `block_id` 发起。

## v2.0.2 — 2026-07-13

### 新增

- 新增快捷房间配置面板，支持输入短号或正常房间号。
- 新增房间号一键验证，可在不连接直播间的情况下查看主播、真实房间号、直播标题和开播状态。
- 新增快捷房间卡片，可从未连接状态直接进入常用直播间。
- 新增“一键添加当前房间”，将正在管理的直播间保存为快捷入口。
- 快捷房间信息持久化到本机 `config/quick_rooms.json`，并提供配置示例文件。

### 改进

- 新版金毛骑士产品图标替换原有文字占位标识，并同步用于登录页、工作台顶栏和浏览器标签页。
- 一键启动器会在 5173 被占用时自动尝试 5174、5175 等后续端口，并打开实际地址。
- 完善电脑版竖屏布局，适配旋转显示器和 Windows 缩放场景。
- 将“弹幕与房管名单正在实时同步”修正为“弹幕与禁言名单正在实时同步”。
- 修复深色模式下快捷房间验证结果中的真实房间号和直播标题对比度不足的问题。
- 修复普通弹幕毫秒时间戳被重复放大、导致发送时间大幅跳变的问题。

### 安全与约束

- 快捷房间测试只调用房间信息解析接口，不会连接直播间或执行任何房管操作。
- 快捷房间初版不提供删除 API 或页面删除按钮；删除记录需要停止程序后手动编辑配置文件。
