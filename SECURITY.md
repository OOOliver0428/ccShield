# 安全策略

ccShield 会在本机保存 B站登录 Cookie，并具备禁言、解禁等房管能力。请勿在公开 Issue、Pull Request、日志、截图或测试数据中提交任何真实凭据、二维码或个人房间配置。

## 支持范围

安全修复优先覆盖当前主线和最新发布版本。过旧版本可能只提供升级建议。

## 报告安全问题

1. 优先使用仓库 **Security** 页面提供的私密漏洞报告入口。
2. 如果私密报告入口不可用，请先通过维护者的 GitHub 主页建立联系，不要在公开内容中披露漏洞细节或凭据。
3. 报告中请包含受影响版本、影响范围、最小复现步骤和已执行的脱敏处理。

如果已经意外公开 Cookie，请立即在 B站侧退出相关登录或重新扫码轮换登录态，并删除公开内容；不要等待代码修复后再处理。

## 安全边界

本项目默认只监听回环地址，面向单机单用户使用，不应直接暴露到局域网或公网。完整的威胁模型、凭据存储约束和测试安全规则见 [安全模型](docs/security.md)。
